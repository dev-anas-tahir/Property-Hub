# Django Cotton Tailwind 4 No Node Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move Property Hub to Django-owned UI infrastructure: `django-cotton`, `django-tailwind-cli`, Tailwind CSS 4, DaisyUI, CDN HTMX/Alpine, and no Node.js project dependency.

**Architecture:** Django stays server-rendered with normal page/layout templates. Reusable UI fragments move to Cotton components under `templates/cotton/` and `apps/properties/templates/cotton/properties/`. Tailwind CSS 4 is built by `django-tailwind-cli` from `assets/css/input.css` into the existing runtime file `static/dist/output.css`.

**Tech Stack:** Django 6, Python 3.13, `uv`, `django-cotton==2.7.2`, `django-tailwind-cli==4.6.2`, DaisyUI-enabled `tailwindcss-extra` CLI `2.8.3` (Tailwind CSS 4.2.2 + DaisyUI 5.5.19), HTMX 1.9.10 CDN, Alpine.js 3.13.3 CDN.

---

## Decisions Already Made

- Single branch switches both packages.
- Migrate to Tailwind CSS 4.
- Remove Node.js entirely.
- Load HTMX and Alpine.js from pinned CDN URLs with SRI.
- Delete `frontend/`.
- Use `assets/css/input.css` as Tailwind source.
- Keep DaisyUI and preserve the current `propertyhub` theme.
- Convert reusable components only; keep pages, layouts, and page-like HTMX partials as normal Django templates.
- Use Cotton default folders: shared components in `templates/cotton/`, property-specific components in `apps/properties/templates/cotton/properties/`.
- Use Cotton HTML-like `<c-...>` syntax.
- Keep the explicit form-field prop API.
- Update all project-facing docs and commands that mention Node, Tailwind, or `frontend/`.
